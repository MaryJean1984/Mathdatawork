# 词汇大世界 (MathDataWork) - 多语种智能单词记忆平台

本项目是一个高标准、硬核的数据库实训项目，以 **MySQL 8.0** 为核心，结合 C++ (Crow 框架) 后端与 Vue3 科技感前端，完整实现了多语种单词背诵、自律模式打卡、个人信息管理等核心业务。

## 🎯 核心数据库亮点 (满足高分评优要求)

本项目在数据库层面进行了极具深度的工程化升级，**显著使用了 MySQL 数据库**的各种高阶特性，绝不仅限于简单的增删改查：

### 1. 完整的增、删、查、改 (CRUD) 业务闭环
- **用户管理**：通过 `sys_user` 表实现了用户昵称、密码、手机号、学习目标等信息的完整录入（Insert）、查询（Select）、修改（Update）与删除（Delete）闭环。
- **单词与记录管理**：实现了单词词库、错题本、待背库的完整关联查询与状态更新。

### 2. SQL 语句优化与千万级深分页 (SQL Optimization)
- **延迟关联 (Deferred Join)**：在 `backend_cpp/main.cpp` 的分页接口中，针对可能出现的千万级海量词库深分页问题，**放弃了传统的 `LIMIT offset, size`**。采用了 `INNER JOIN` 子查询先走主键索引定位 ID，再回表查全字段的优化方案，大幅降低磁盘 I/O。
- **MySQL 原生 JSON 字段**：利用 MySQL 8.0 的 `JSON_MERGE_PATCH` 等函数，直接在 SQL 层面高效合并 AI 生成的复杂非结构化数据（同义词、例句等），避免了低效的先查后写。

### 3. 索引优化与覆盖索引 (Index Optimization)
- 在 `db/schema.sql` 和 `db/v2_advanced_features.sql` 中，为高频查询字段（如 `username`, `phone`, `status`）建立了精准的单列及复合联合索引。
- **零回表优化**：特别设计了 `idx_lang_stage_name` 联合覆盖索引，使得后台管理系统在根据条件筛选用户时，无需回表即可直接在索引树上拿到结果 (`Using index`)。

### 4. 数据安全与冷热分离 (AES & Archiving)
- **AES 对称加密**：对于用户的敏感数据（如手机号），在 SQL 层直接调用 `AES_ENCRYPT` 和 `AES_DECRYPT` 进行落库和查询，防止数据库被拖库后信息泄露。
- **表分区与归档**：使用 `RANGE PARTITION` 按月对学习记录表进行物理分区；并编写了 MySQL `EVENT` 定时任务，每天自动将 180 天前的冷数据归档至历史表。

### 5. 触发器全自动审计 (Audit Triggers)
- 编写了 `AFTER UPDATE` 触发器，一旦用户关键信息发生变更，数据库底层会自动记录修改前后的快照并存入审计日志表，保证数据的强一致性溯源。

## 🤖 AI 智能助手：“小词” (Xiao Ci)

在前端界面右下角，我们深度集成了一位名为 **“小词”** 的全局 AI 智能助手：
- **专属形象**：小词的形象是一只由 AI 实时生成的**卡通画眉鸟**（基于赛博科技风格设计）。
- **个性化称呼与打招呼**：小词会读取用户在“个人资料”中自定义修改的昵称，并用这个专属称呼热情地向用户打招呼（如：“你好，CyberStudent！我是小词，你的专属学习助手！”）。
- **实时鼓励与自律监督**：小词会实时监控用户的“自律模式”打卡进度，并智能提示：“今天还要背 X 个单词才能完成自律目标哦，冲鸭！”，在练习时也会不断给用户加油打气。

## 💻 运行说明

1. **前端启动**：
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
2. **后端编译**：
   使用 Visual Studio C++ (MSVC) 工具链，进入 `backend_cpp` 目录运行 `run_backend.bat` 编译启动。
3. **数据库脚本**：
   请依次在 MySQL 8.0 环境中执行 `db/` 目录下的 `schema.sql`, `v2_advanced_features.sql`, `v3_database_engineering_upgrade.sql`。
